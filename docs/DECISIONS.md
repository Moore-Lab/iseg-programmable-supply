> instrument does **not** pass smoothly through zero, it **switches** polarity.
> **"unipolar / dual-output"** = mode 2, two outputs, both energisable at once.
> Rows written before A5 say "BIPOLAR COMBINED" for mode 1; read that as **pseudo-bipolar**.

## 3. Control architecture — set path, monitor path, interlock

`docs/CONTROL_ARCHITECTURE.md` + `docs/studies/control_arch_numbers.py`, `control_arch_divider.py`.

| ID | Decision | Rationale | Evidence | Status |
|---|---|---|---|---|
| ARCH-01 | **No open-loop accuracy better than 1 % is physically obtainable**, regardless of DAC quality, because `Vout/Vnom = Vset/Vref` and the module's internal Vref is ±1 % | The central architectural result. It means the independent output monitor is **the only route to sub-1 % accuracy**, not merely a safety item — and it converts every static set-path error into a one-time calibration constant | `[verified-run]` | **frozen** |
| ARCH-02 | Set path is an **external DAC + precision reference**, not PWM | The case against PWM is **not speed** (the module's own 100 ms pole makes the filter penalty only +10 %) — it is the **reference**: a PWM DAC's full scale *is* the 3.3 V rail (±1–3 %, drifting with WiFi TX bursts) vs a precision reference at ±0.05 % | `[verified-run]` | **frozen** |
| ARCH-03 | **12-bit set path, 16-bit monitor path** | LSB = Vnom/2^bits exactly, *provided* the DAC references the same voltage as the module Vref.
